# Contributing Guidelines

## How to Contribute

### 1. Choose Your Chapter

Browse the [11 chapters](./README.md#chapters) and select one that interests you or aligns with your expertise.

### 2. Pick a Subsection

Each chapter has three subsections:
- **tools/** - For security tools, frameworks, and software
- **best-practices/** - For guidelines, standards, and recommended approaches
- **articles/** - For research papers, blog posts, tutorials, and external resources

### 3. Create Your Content

#### File Naming Convention

Use descriptive, kebab-case file names:
- Tools: `tool-name.md` (e.g., `semgrep.md`, `snyk.md`)
- Best Practices: `topic-name.md` (e.g., `secure-password-storage.md`)
- Articles: `descriptive-title.md` (e.g., `owasp-top-10-explained.md`)

#### Content Structure

While we don't enforce strict templates, good contributions typically include:

**For Tools:**
- What it does and why it's useful
- Key features
- Installation/setup instructions or links
- Usage examples (when applicable)
- Pros and cons
- Links to official documentation
- License information

**For Best Practices:**
- The problem or challenge being addressed
- The recommended approach
- Why this approach works
- Examples or code snippets
- Common pitfalls to avoid
- References and further reading

**For Articles:**
- Brief summary of the content
- Why it's valuable
- Key takeaways
- Author and publication date
- Direct link to the article
- Relevant tags/topics
